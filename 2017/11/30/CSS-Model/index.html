CSS中盒模型基本概念

发表于 2017-11-30 | 分类于 CSS

原理

css 属性：Box Moel分为两种：W3C标准和IE标准盒子模型。
大多数浏览器采用W3C标准模型，而IE中采用Microsoft自己的标准。
怪异模式是“部分浏览器在支持W3C标准的同时还保留了原来的解析模式”，怪异模式主要表现再IE内核的浏览器。
当不对Doctype进行定义时，会触发怪异模式。

在标准模式下，一个块的总宽度=width+margin(左右)+padding(左右)+border(左右)
在怪异模式下，一个块的总宽度=width+margin（左右）（既width已经包含了padding和border值）

可以看的出来IE盒子的width是包含了border的宽度.而标准盒子的width没有包含border的宽度.

这里我们针对标准盒子进行概念讲解:
如果把上图看做HTML里的一个标签div盒模型

OK!来戏弄一番这个盒子:

  • div中的内容content就是图片中橘黄色的区域.
  • 只有内容才会有width,height属性,不包含边框!
  • 边框(border)就是图片中深蓝色的区域,它具有粗细的属性,border-top,border-bottom,border-left,border-right代表了上下左右的粗细程度.
  • 内容到边框的距离就是padding,也就是内边距,即黄色区域.
  • 最后就是margin,即外边距(原谅色区域), 它代表着<div>与旁边的元素或者是父元素的距离,margin-top,margin-bottom,margin-left,margin-right代表了上下左右的粗细程度.


第一个红色div标签我给了它padding:20px;第二个黄色div标签给了它padding:0px;从图中明显可以看出2个div边框离文本的距离大小不同，也发现了2个div大小也不一样（padding撑开了容器），这就是padding，简称内边距。


图上面那条绿色作为基点，第一个红色div给了margin:20px;大家可以看到红色div到绿色的线有了一定的距离，红色div离黄色div也有一定的距离，这个距离就是margin:20px，简称外边距。
